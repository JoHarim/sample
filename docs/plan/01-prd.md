# PRD — 스마트 알람 (가칭)
> 작성일: 2026-07-05 · blueprint 인터뷰 기반 · 상태: 확정(초안)
> 이름은 아직 미정 — 가칭 "스마트 알람"

## 1. 개요
- 한 줄 정의: 아침에 잘 못 일어나는 사람을 위해, 날씨에 맞춰 알람 시각이 스스로 바뀌고(똑똑한 시간) 무심코 못 끄게 하는(확실히 깨우기) 모바일 알람 앱.
- 레퍼런스: 알라미(Alarmy). 다른 점: 알라미는 "미션"으로 깨우지만 시각은 고정 — 우리는 상황(날씨)에 맞춰 시각 자체를 조정한다.
- 배경·동기: 만든 사람 본인이 아침에 잘 못 일어남. 내 문제부터 푼다. 첫 사용자 = 나.
- 성공 기준: 출시 후 100명이 설치한다.

## 2. 문제·타겟
- 페르소나: 아침 기상이 힘든 20~30대 직장인·학생 (= 만든 사람 본인). 스마트폰에 익숙.
- 문제 장면: 알람을 여러 개 맞춰도 무의식중에 꺼버리고 늦잠. 미션 알람을 써봐도 알람 시각이 고정이라 매일 상황과 안 맞음.
- 현재 대안과 한계: 알라미 같은 미션 알람 사용 중 / 한계 = 미션을 풀어도 시각이 고정이라 날씨·상황을 반영 못 함.
- 사용자 vs 결제자: 같음(본인). 초기 무료. · 규모 가늠: 3개월 지인 중심 → 출시 후 100명.
- 역할: 사용자(본인 알람 관리) · 운영자(나 — 사용자 수 모니터링만). 관리자 페이지는 MVP에 없음.
- 민감정보 판정: 수면 패턴·위치는 민감정보 → 최소화. MVP는 수면 미수집, 위치는 "도시명"만 사용하고 폰 로컬에만 저장(서버 미전송). 날씨 API엔 도시명만 전송.

## 3. 기능 (MVP)
| 순위(빌드 순서) | 기능 | 완료 기준 |
|---|---|---|
| 1 | 기본 알람 (토대) | 시간·요일을 정해 알람을 만들면, 그 시각에 폰이 잠겨 있어도 소리·진동 알림이 온다 |
| 2 | 확실히 깨우기 (밀어서 끄기 + 큰 소리) | 알람이 울리면 화면을 끝까지 밀어야 꺼지고, 큰 소리·진동으로 무심코 못 끄게 한다 (스누즈 없음) |
| 3 | 날씨 따라 조정 | 비/눈 예보가 있으면 설정 시각보다 자동으로 N분 일찍 울리고, 조정된 시각을 목록에 미리 보여준다 |

### 완성 제품 로드맵 (꿈은 보존, 층으로 쌓는다)
- 다음 버전 (감당 가능, 이번엔 안 함): 교통·출근길 따라 조정 / 로그인·기기 동기화 / 간단 미션 / 스누즈 제한 / 소리·진동 커스텀 / 기상 기록·통계 / 캘린더 첫 일정 연동.
- 완성 제품 (최종 그림): 날씨·교통·일정·수면을 모두 종합해 "오늘 최적 기상 시각"을 스스로 정해주는 개인 기상 비서.
- 감당선 밖 (첫 앱 스택으로는 무리): 실제 수면 감지 스마트웨이크(밤새 센서로 수면단계 추정 = 배터리·백그라운드·정확도 문제, 전문 영역). 대체 경로 = (1) 직접 공부: expo-sensors 가속도계 / 수면 추적 알고리즘 / 애플 HealthKit·Google Fit 연동, 또는 (2) 전문가 외주. 선택은 나중에 해도 되며 MVP에는 영향 없음.

### Out of Scope (절대 안 함)
- 교통 조정 · 수면 감지 · 로그인/기기 동기화 · 간단 미션 · 스누즈(및 스누즈 제한) · 소리·진동 커스텀 · 기상 통계 · 캘린더 연동 · 서버 저장 · 관리자 페이지.

## 7. 외부 연동
기본 스택 조정: 이 앱은 모바일(Expo) + 폰 로컬 저장이라 MVP에서 Supabase(서버 DB/로그인)·Vercel(웹 배포)은 쓰지 않는다. 배포는 Expo EAS Build → 앱스토어/구글플레이. (동기화가 필요한 다음 버전에서 Supabase 도입.)

| 저장 | 폰 로컬 (Expo SQLite/AsyncStorage) | — | (없음) |

## 8. 지킬 조건 (비기능 — 앱이 하는 일이 아니라 지켜야 할 규칙)
- 권한 등급: 알림 권한 필수(없으면 안 울림 → 목록 상단 경고 배너 + 설정 유도). 위치 권한 불필요(도시 직접 설정).
- 관리자: 없음(개인 앱). 사용자 수는 필요 시 직접 확인.
- 최우선 보호 데이터: 알람·설정은 폰 밖으로 나가지 않음. 날씨 API엔 도시명만 전송.
- 실패 시 안내 / 외부 장애 시: 날씨 조회 실패·오프라인이어도 알람은 원래 시각에 반드시 울린다(알람 최우선). 저장 실패 시 재시도.
- 성능: 날씨는 알람 예보 시점 근처에만 조회 → 무료 한도 내.
- 백업: 폰 로컬 저장이라 기기 분실 시 데이터 손실 가능 → 기기 동기화(다음 버전)에서 해결.
