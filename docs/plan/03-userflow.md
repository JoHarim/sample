# 유저 플로우 — 스마트 알람 (가칭)

## 메인 플로우 (첫 실행 → 알람이 울려 끄기까지)
```mermaid
flowchart TD
  A[앱 첫 실행] --> B[S0 온보딩: 알림 권한 요청]
  B --> C[도시 설정]
  C --> D[S1 알람 목록/홈]
  D -->|+ 버튼| E[S2 알람 추가]
  E --> F{저장 유효성}
  F -->|시간·요일 OK| G[폰 로컬에 저장]
  F -->|미입력| E
  G --> D
  D -->|설정 시각 도달| H{날씨 조정 켜짐?}
  H -->|예 · 비/눈 예보| I[N분 일찍 예약]
  H -->|아니오/맑음/조회 실패| J[원래 시각]
  I --> K[S3 알람 울림]
  J --> K
  K -->|밀어서 끄기| L[종료 - 다시 안 울림]
```

## 재방문 플로우
앱을 열면 바로 S1 목록. 온보딩은 없음(첫 실행 1회). 알람 추가·수정·삭제·켜고 끄기.

## 운영자 플로우
없음(MVP). 관리자 페이지 없음. 사용자 수는 필요 시 스토어/집계로 직접 확인.

## 상태 흐름
알람에는 켜짐/꺼짐 두 상태만. 진행 상태 머신은 없음(스누즈 없음이라 "울림→종료"가 끝).
